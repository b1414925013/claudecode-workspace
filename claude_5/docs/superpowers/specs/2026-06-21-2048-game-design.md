# 2048 网页游戏 — 设计文档

## 概述

基于 Vue 3 + Vite + TypeScript 的 2048 数字合并游戏，单页面应用，零后端依赖。支持多棋盘尺寸、深色模式、触屏/键盘操作、游戏状态持久化。

## 技术栈

| 层级 | 技术 |
|------|------|
| 构建工具 | Vite 5 |
| 框架 | Vue 3 (Composition API) |
| 语言 | TypeScript |
| 样式 | Scoped CSS + CSS 变量 |
| 存储 | localStorage |
| 包管理 | npm |

## 组件设计

### App.vue

根组件，组合所有子组件，通过 provide/inject 向下传递游戏状态和方法。

### GameHeader.vue

- 显示游戏标题 "2048"
- 当前分数（带增减动画效果）
- 最高分（按棋盘尺寸分别存储）
- "新游戏" 按钮

### GameBoard.vue

- 渲染 `n×n` 网格（`display: grid`）
- 监听 `touchstart` / `touchmove` / `touchend` 计算滑动方向
- 动画锁定期间屏蔽输入
- 自动保存到 localStorage（每次移动后触发）

### GameTile.vue

- 接收位置 `(row, col)`、数值、是否为新建/合并标识
- 使用 CSS `transform: translate()` 实现平滑移动
- 新建块：`scale(0) → scale(1)` 弹出动画
- 合并块：`scale(1) → scale(1.2) → scale(1)` 脉冲动画
- 100ms+ 防连按锁

### GameControls.vue

- "撤销" 按钮（最多可连续撤销 10 步）
- 棋盘尺寸下拉选择器（4×4 / 5×5 / 6×6），切换时重置游戏
- 深色模式切换按钮（🌙/☀️）

### GameOverlay.vue

- 胜利弹窗（显示"恭喜！你赢了！" + "再来一局" 按钮）
- 失败弹窗（显示"游戏结束" + 最终分数 + "再来一局" 按钮）
- 半透明遮罩层，阻止误操作

## 核心逻辑

### 类型定义 (`types/game.ts`)

```typescript
export interface TileData {
  id: number       // 唯一标识，用于 Vue key 和动画追踪
  value: number
  row: number
  col: number
  isNew?: boolean      // 是否新生成的（触发弹出动画）
  isMerged?: boolean   // 是否刚合并的（触发脉冲动画）
}

export type BoardSize = 4 | 5 | 6
export type Direction = 'up' | 'down' | 'left' | 'right'
export type GameStatus = 'playing' | 'won' | 'lost'

export interface GameState {
  board: (TileData | null)[][]   // 二维数组
  score: number
  bestScore: Record<BoardSize, number>
  status: GameStatus
  boardSize: BoardSize
  history: { board: (TileData | null)[][]; score: number }[]
  isDarkMode: boolean
}
```

### 游戏状态 `useGameState.ts`

- 提供响应式 `GameState`
- `initGame(size)` — 初始化/重置棋盘
- `undo()` — 从历史栈恢复上一步状态
- `saveToStorage()` / `loadFromStorage()` — localStorage 序列化/反序列化

### 核心算法 `useGameLogic.ts`

```
移动流程（以"左移"为例）：
1. 保存当前快照到历史栈（最多 10 步）
2. 对每一行，从左到右执行：
   a. 移除中间空位（压缩）
   b. 相邻同值合并（右侧值 ×2，左侧置空）
   c. 再次压缩
3. 随机选择一个空位生成新块（90% 概率 2，10% 概率 4）
4. 检查胜负条件
5. 更新分数
6. 动画锁定 100ms
```

关键函数：
- `move(direction)` — 对外唯一接口
- `slideRow(row)` — 单行滑动合并
- `generateTile()` — 空位随机生成
- `checkWin()` — 是否存在 ≥2048 的块
- `checkLose()` — 是否无空位且无合并可能
- `canMove()` — 棋盘是否还可操作

### 本地存储 `useStorage.ts`

```typescript
interface StorageData {
  board: SerializedTile[]
  score: number
  boardSize: BoardSize
  bestScore: Record<BoardSize, number>
}
```

- 键名：`'2048-game-state'`, `'2048-best-score'`, `'2048-dark-mode'`
- localStorage 不可用时 try/catch 捕获异常，静默降级
- 每次移动后自动保存当前进度

## 视觉设计

### 布局

- 居中卡片式布局，最大宽度 500px
- 分数区域在标题右侧并列显示
- 控制栏在棋盘下方紧凑排列
- 棋盘内单元格间距 8px（gap），圆角 6px

### 动画规格

| 动画 | 时长 | 缓动函数 |
|------|------|---------|
| 方块移动 | 100ms | ease |
| 新建块弹出 | 200ms | ease-out |
| 合并脉冲 | 200ms | ease-in-out |
| 分数增减 | 150ms | ease |

### 深色模式

- 通过 CSS 变量切换，`<html>` 标签添加 `.dark` class
- `prefers-color-scheme: dark` 媒体查询检测初始偏好
- 用户手动切换后覆盖媒体查询结果并存入 localStorage

### 响应式断点

| 屏幕宽度 | 棋盘行为 |
|---------|---------|
| ≥500px | 固定尺寸，居中显示 |
| <500px | 宽度自适应（100% - 2×16px 边距） |
| 触屏设备 | 棋盘高度按宽高比 1:1 设置 |

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| localStorage 不可用 | try/catch 捕获，游戏正常运行不保存 |
| localStorage 数据损坏 | 解析失败时清空存档，重新开始 |
| 极快连按 | 动画锁定期间忽略输入，不堆叠操作 |
| 浏览器窗口极小 | 最小棋盘尺寸 280×280，内容溢出时缩小字号 |
| 触屏误触 | 仅当滑动距离 > 20px 才触发移动，忽略轻触 |

## 非功能需求

- **性能**：每次移动操作在 1ms 内完成计算
- **兼容性**：支持 Chrome / Firefox / Safari / Edge 最新版本
- **无外部依赖**：不引入任何第三方 UI 库或游戏库
- **可访问性**：棋盘使用 `role="grid"`，方块使用 `role="gridcell"`，支持 `aria-label`
