# 浏览器自动化：Playwright CLI + Skill 搭建 AI 自动化框架

> 告别一切重复枯燥任务，让 AI 自动操作浏览器，省时省 TOKEN。
>
> 视频版：[Bilibili](https://www.bilibili.com/video/BV1ooDyBmE6v/)

## 概述

这套流程可以让 AI 自动操作浏览器，代替你完成一切机械重复性的工作，而且特别省 TOKEN。很多工作流甚至全程不需要 AI 参与，**0 TOKEN** 就能把自动化任务跑起来。

例如：
- 0 TOKEN 抓取电商网站评论，导出成 CSV 文件
- 自动把 Markdown 文章发布到 X (Twitter)
- 对自己开发的 Web App 进行 AI 自动化测试

我们并不需要懂浏览器的相关知识，只用自然语言就能完成这些任务。

本期视频使用的 Agent 框架是 **Claude Code** 或 **Codex**，浏览器自动化方案是 **Playwright CLI** 搭配配套的 **Skill**。

Playwright CLI 是 26 年初微软开源的全新浏览器自动化工具。

- **GitHub**: <https://github.com/microsoft/playwright-cli>
- 根据官方基准测试，Playwright CLI 比起传统 Playwright MCP 方案，能够减少约 **4 倍**的 TOKEN 消耗

工具搭建好以后，可以把很多固定的工作流程沉淀成 Skills，让 AI 能够又快又省地完成任务。熟练后你会发现很多固定流程甚至不需要 AI 参与，只需让 AI 编写好一个固定的脚本，就可以 **0 TOKEN** 全自动完成工作。

## 基础使用

### 环境准备

1. 确保电脑上安装了 **Node.js**（[下载地址](https://nodejs.org/en/download)）
2. 打开命令行终端，安装 Playwright CLI：

```bash
npm install -g @playwright/cli@latest
```

3. 确保电脑上安装了 **Chrome 浏览器**（Edge 也可以，但推荐 Chrome）

### 验证安装

使用 Playwright CLI 操作 Chrome 打开网页：

```bash
playwright-cli open google.com --headed
```

- `--headed`：使用**有头浏览器**（能看到浏览器界面）
- 不加该参数则默认使用**无头浏览器**，在后台静默运行

Playwright CLI 只会输出简洁的网页摘要，而不是返回整个网页的完整 DOM 结构，并附带一个网页快照文件地址。AI Agent 按需读取该文件获取详细信息，这比 MCP 把全部内容塞入上下文要节省得多。

### 截图

```bash
playwright-cli screenshot
```

截图以 PNG 文件形式存放在本地磁盘，由 AI 决定是否读取，而非像 MCP 那样直接把图片塞入 AI 上下文。

### 持久化状态

```bash
playwright-cli open google.com --headed --persistent
```

`--persistent` 把 Cookie、登录状态、本地存储等数据写到磁盘，下次使用时无需重新登录。

## 接入 AI Agent

Playwright CLI 是一个新工具，AI 并不知道如何使用。需要给 AI 搭配 **Skills**（技能说明文档）一起使用。

> Playwright CLI 作为技术底座，Skills 作为说明文档，**CLI + Skills** 取代传统 MCP 方式，这是最新的技术趋势。

### 安装 Skills

```bash
playwright-cli install --skills
```

Skills 会安装到当前项目目录下。

### Claude Code 接入

在项目目录启动 Claude Code，询问"你有哪些 skills"，AI 能成功读取到 Playwright CLI 技能即为配置成功。

### Codex 接入

将存放 Skills 的文件夹名从 `.claude` 改成 `.codex` 以适配 Codex。在 Codex 中输入 `/skills`，看到列出 Playwright CLI 即为配置成功。

## 进阶实战

### 案例：抓取商品评论

让 AI 查看商品前 100 条评论并保存到 CSV 文件。

**第一次运行**：AI 自己探索、试错，用掉了 **41%** 的上下文窗口，最终成功完成任务。

#### 优化方式一：提炼为 Skill

让 AI 把过程总结保存成 Skill：

> 创建一个新的 skill，把刚才打开网站、查看评论、保存评论的全过程以及遇到的坑都提炼出来，保存到 skills 中。后面我只要让你保存评论，你就能调用这个 skill 完成任务。

**效果对比**：
- 第一次（无 Skill）：41% 上下文 → 完成任务
- 第二次（有 Skill）：**5%** 上下文 → 完美完成任务
- 效率提升约 **10 倍**

#### 优化方式二：编写固定脚本

对于固定流程，无需 AI 智能控制，直接编写脚本：

> 把刚才所有的 playwright cli 命令汇总成一个脚本，执行脚本就能获取商品的前 100 条评论并保存到 CSV 文件。注意每一步都要有合理的延时与等待，确保任务成功。脚本写完先测试一轮。

AI 以编程方式把步骤固定为脚本（如 PowerShell 脚本），无需 AI 参与，**0 TOKEN** 完成任务。

### 自动化流程总结

```
准备阶段 → 安装 Node.js、Playwright CLI、Chrome
     ↓
选择 AI Agent，安装 Playwright CLI Skill
     ↓
给 AI 复杂任务，让它自己摸索执行
     ↓
将执行过程提炼为 Skill（避免重复踩坑）
     ↓
重试相同任务 → Token 消耗降低约 10 倍
     ↓
（可选）将固定流程编写为脚本 → 0 Token 全自动
```

## 实战案例：自动发布文章到 X (Twitter)

### 背景

将 Markdown 格式的笔记文章发布到 X 平台时遇到问题：
- Markdown 格式粘贴会错乱，图片无法显示
- HTML 格式可以粘贴，但图片会变成照相机图标占位符
- 需要手动删除占位符、逐一粘贴图片，非常繁琐

### 第一步：转换文章格式

```prompt
帮我编写一个 Python 脚本，把文章里的图片下载到本地，从 001 开始编号，放到一个文件夹中，
先转换成一个只使用本地图片的 Markdown 文章，然后运行 pandoc 把本地图片的 Markdown 文章
转换成 html 格式。注意 html 格式里每张图都应该是独立段落。
```

AI 编写脚本并完成转换，图片存入 `images` 文件夹，文章转换为 HTML 格式。

### 第二步：AI 自动发布

```prompt
使用 playwright CLI，先打开这个网站，创建一个新的文章，然后把这个 html 的内容粘贴进去，
找到所有的照相机小图标的位置，先按退格键删除掉小图标，然后在图片文件夹中找到图片，
先复制出来，按 Ctrl+V 粘贴进去。小 icon 的数量与图片数量相等，需要按顺序替换。
```

AI 打开浏览器、创建草稿、粘贴 HTML、逐一替换图片，整个过程自动完成。

### 第三步：固化流程为 Skill

让 AI 将全流程整理成 Skill，后续只需提供文章路径即可自动发布。

> Skill 源码参考：<https://github.com/tech-shrimp/x-article-auto-publisher-skill>

## AI 自动化测试

对自行开发的 Web App 进行自动化测试：

```prompt
阅读代码，把从注册开始的主体流程写一个中文的测试文档，然后再用 playwright CLI 打开网页，
根据你的测试用例完成测试。
```

流程：
1. AI 阅读代码学习项目功能
2. 编写测试文档（注册 → 登录 → 创建简历 → 上传等步骤）
3. 使用 Playwright CLI 打开浏览器自动执行测试
4. 输出测试结论

还可以要求 AI 编写更多测试用例，进行全自动测试。配合 **OpenClaw** 等带定时任务的 Agent 框架，定时对系统进行回归测试，一旦出现 Bug 就能自动发现并通知，大幅节省人工测试成本。

Playwright CLI + Skill 的浏览器自动化方案，相比传统 MCP 方式：
